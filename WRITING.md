# Writing

How agents draft in this repo. Not a second brand voice system.

## Authority

| Layer | Role |
| --- | --- |
| [`ai/skills/writing-standard`](ai/skills/writing-standard/SKILL.md) | Full workflow: classify, diagnose, rewrite, self-check. |
| [`reference/voice.md`](reference/voice.md) | Tone, banned words, zero em dashes. |
| This file | Portable self-check when you only need the short list. |

Prefer the skill. Fall back to the self-check below. Do not invent a third standard.

## Scope in this repo

Applies to: `README`, `GETTING_STARTED`, `CONVENTIONS`, `AGENTS.md`, `playbook/`, `templates/`, `world/`, `projects/`, `ai/` (non-skill docs), `examples/`.

Does not rewrite: `theory/` (archived research). Cite theory; do not match its prose.

Studio bible docs are reference procedures. Apply the self-check (no em dashes, no slop) even when the skill's creative-fiction carve-out would otherwise skip.

## Always apply

1. Lead with the point. Context after.
2. Active voice in decision and rule sentences.
3. Specific over vague: name paths, owners, versions, counts.
4. One idea per sentence when writing rules.
5. No em dashes. No en dashes. Use a hyphen, period, or colon.
6. No rhetorical questions in checklists. Use imperatives.
7. No mission fluff: inspire, empower, elevate, reimagine, journey (as filler).

## Self-check before merge

- [ ] Lead states the point
- [ ] Zero em dashes / en dashes
- [ ] Zero banned words from [`reference/voice.md`](reference/voice.md)
- [ ] Claims name a path, owner, number, or get cut
- [ ] `projects/` cites `world/` paths (no lore paste)
- [ ] Pub test: would you say this to a studio teammate

## Theory contamination

Do not copy treatise cadence, hedge stacks, or academic padding into `world/`. Treatise is evidence and case cites only.
